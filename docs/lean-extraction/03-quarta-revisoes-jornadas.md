# Quarta-feira — Revisões Técnica/Negócio/UX e Jornadas

## Revisões das Features

Para cada funcionalidade é atribuído um valor de percepção em três dimensões:

- **Esforço (E):** Qual o tamanho do esforço para fazer essa funcionalidade? (E = pequeno, EE = médio, EEE = grande)
- **Negócio ($):** O quão necessária é essa funcionalidade para o negócio? ($ = baixo, $$ = médio, $$$ = alto)
- **UX (❤️):** O quanto as personas irão "amar" essa funcionalidade? (❤️ = baixo, ❤️❤️ = médio, ❤️❤️❤️ = alto)

Também são avaliados dois eixos de clareza:
- **How to do:** Sabemos como fazer? Temos ideia dos requisitos para implementar?
- **What to do:** Está claro o resultado final esperado? Como medir sucesso? Como seria a interação do usuário?

## Jornadas

São uma série de passos que leva o usuário a atingir o seu objetivo. Pode abranger tarefas cotidianas, de trabalho e/ou sentimentos ao longo do dia.

### Jornada da Persona 1 (Letícia / De Marco)

As etapas identificadas na jornada do perito/compliance:

1. **Acionamento** — recebimento da demanda
2. **Apreciação do Caso** — entender o escopo e riscos
3. **Análise de Histórico** — cruzar dados e logs disponíveis
4. **Coleta de evidência** — preservar e catalogar as evidências encontradas
5. **Construção do Laudo** — produzir o documento técnico final
