# Lean Inception — Visão Geral

Proposta de Paulo Caroli que simplifica inceptions pré-existentes, buscando foco no MVP (o quê fazer), postergando detalhes como estimativas de histórias, arquitetura e afins para outro momento (como fazer).

## Participantes recomendados

Todos os envolvidos no produto (time + stakeholders). Alguns podem participar apenas do kick-off e do show case.

- Especialista de Domínio
- Time de produto (PO, PM ou figuras semelhantes)
- Agilistas (coordenadores, staff)
- Time de desenvolvimento (Arquiteto, Tech Lead, devs)
- Designers (frequentemente atuam como facilitadores)

## Agenda da semana

| Dia    | Manhã                                          | Tarde                              |
|--------|------------------------------------------------|------------------------------------|
| Segunda | Introdução, Kick off, Visão do produto        | É/Não É — Faz/Não Faz             |
| Terça  | Personas                                       | Features                           |

| Quinta | Alinhar features e jornadas                    | Sequenciar features                |
| Sexta  | Construir o MVP Canvas                         | Show case da inception             |
